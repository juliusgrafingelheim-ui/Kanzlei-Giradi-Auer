## 📋 Feldtypen im Storyblok:

| Feldtyp | Verwendung |
|---------|-----------|
| **text** | Überschriften, Titel, kurze Texte, Icons (als Name), URLs |
| **textarea** | Längere Texte, Beschreibungen, Adressen (max 160 Zeichen für SEO) |
| **richtext** | Impressum, Datenschutz, Team-Bios, Geschichte - WYSIWYG Editor! |
| **asset** | Alle Bilder - Team-Fotos, Hero-Images, SEO-Images |

### 2. **Preview Token für Website:**
Nach dem Setup brauchst du den Preview Token für deine Website:

**Lokal (.env Datei):**
```env
VITE_STORYBLOK_TOKEN=dein-preview-token
```

**Vercel (Environment Variables):**
1. Vercel Dashboard → Dein Projekt
2. Settings → Environment Variables
3. Add: `VITE_STORYBLOK_TOKEN` = `dein-preview-token`
4. Alle 3 Environments auswählen
5. Redeploy

## 📖 Verwendung im Frontend:

```tsx
import { useStoryblok } from "../hooks/useStoryblok";

export function HomePage() {
  const { content, loading } = useStoryblok("home", {
    // Fallback-Daten (aktueller Inhalt)
    hero_title: "Ihr Recht ist unsere Berufung",
    hero_subtitle: "Professionelle Rechtsberatung...",
    seo_title: "Rechtsanwalt Innsbruck | Girardi & Auer",
    // ... alle anderen Felder
  });

  return (
    <>
      <Helmet>
        <title>{content.seo_title}</title>
        <meta name="description" content={content.seo_description} />
      </Helmet>

      <section>
        <h1>{content.hero_title}</h1>
        <p>{content.hero_subtitle}</p>
        <img src={content.hero_image?.filename} alt="Hero" />
      </section>
    </>
  );
}
```

**Dual-Mode:** Funktioniert MIT und OHNE Storyblok! Automatischer Fallback!
